• git config --global user.name для додавання імені користувача
  • git config --global user.email для додавання пошти користувача
  • git init для для запуску нового сховища
  • git status перераховує всі файли, які потрібно зафіксувати
  • git add додає файли до проміжної області
  • git commit для фіксації змін у файлах
  • git log для відображення історії змін для поточної гілки
  • git checkout для перемикання з однієї гілки на іншу
  • git branch для перевірки списку і створення нових локальних гілок
  • git push надсилає внесені зміни гілок до локального сховища
  • git pull отримує та об’єднує зміни з віддаленого сховищі у робочий каталог
  • git merge об’єднує історію вказаної гілки з поточною гілкою
  • git clone для отримання сховища з існуючої URL-адреси
  • .gitignore для додавання списку файлів, які мають бути ігноровані
  • git revert для запису нових комітів, щоб скасувати дію деяких попередніх
  • git reset для скасування комітів та збереження змін
  • git rebase для зміни одного коміта в основі гілки на інший
  • git cherry-pick дозволяє вибрати коміт з однієї гілки та застосувати його до іншої
  • git stash для того, щоб тимчасово відкласти зміни і застосувати їх пізніше
